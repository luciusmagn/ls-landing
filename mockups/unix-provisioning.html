One command, one machine

There is no dashboard to click through and no wizard to survive. You name an image and a time to live; Coldstart hands back a hostname. When the TTL runs out, the machine and its disk cease to exist, which is the point.

  • Images are immutable and versioned; a machine never drifts from its image.
  • Every machine carries a TTL. Renew it deliberately or let it lapse.
  • Snapshots are explicit: coldstart keep, or it never happened.
  • The CLI is the API. Everything below it is plain JSON over HTTPS.
Session transcript
$ coldstart new --image debian-13-slim --ttl 2h
run-9d2a created in 6.8 s
$ ssh dev@run-9d2a.coldstart.example
dev@run-9d2a:~$ uname -sr
Linux 6.12.9-coldstart
dev@run-9d2a:~$ exit
$ coldstart ls
run-9d2a  debian-13-slim  expires in 1 h 54 m
$ coldstart drop run-9d2a
run-9d2a destroyed. billed 312 s
